## 2. Что это за игра (анализ продукта)

**Название:** Jurassic Market  
**Жанр:** настольная экономическая / гонка по кругу (в духе Monopoly), тема Jurassic World.  
**Платформа:** статический веб (HTML + CSS + vanilla JS), без сборщика.  
**Цель партии:** первым накопить **$125 000**.

### Сюжетный каркас
- Чёрный рынок динозавров; цитата **Soyona Santos** (intro-modal) — перед **первым броском** кубика (`introSeen` в состоянии партии); бросок заблокирован, пока не закрыли OK.
- Игроки покупают динозавров, получают доход за полный круг, продают агентам, обмениваются по «привлекательности».

### Режимы
| Режим | Как | Сохранение |
|--------|-----|------------|
| **Офлайн (hot-seat)** | Auth / меню → «Офлайн»; можно добавлять/удалять игроков (2–6) | `localStorage` ключ `jm_campaign_v1` |
| **Обучение** | Меню → «Обучение» | без сохранения; `TUTORIAL_LOCATION` (20 клеток, 1 игрок) |
| **Онлайн** | Меню → создать / войти по коду → лобби комнаты → старт (хост, ≥2) | таблица `games.state` (JSONB) |

Онлайн **не обязателен**: если `js/config.js` пуст — игра полностью работает офлайн.

### Экраны (`state.screen`)
`boot` → `auth` → **`main-menu`** → (`tutorial`/`offline`/`online` game) или **`room-lobby`** → `game`.  
Присоединение к комнате — **только по коду** (способ A): модалка ввода / `?room=CODE`. `JMNet.leaveRoom` удаляет игрока из `room_players`.

### Локации (кампания)
1. `malta` — Мальта, Valletta (эталон, 40 клеток).
2. `biosyn-dolomites` — BioSyn, Доломиты (40).
3. `isla-sorna` — Isla Sorna, Site B (40).
4. `saint-hubert` — Сен-Юбер, лаборатория InGen (40).
5. `san-albertus` — Полуостров Сан-Альбертус (16 клеток по IMG_2106; центр «Катакомбы города» + фото; `boardLayout: { sideH: 3, sideV: 3 }`). События: `return_start`, `fine`, `restaurant` (`pendingAction`: согласие −20% + `eventShield` / отказ), кафе — `rest`.
6. `senegal-lab` — Сенегал, подземная лаборатория (14 клеток по IMG_2107; `boardLayout: { sideH: 3, sideV: 2 }`). Динозавры: Велоцираптор, Тираннозавр, Карнотавр, Аллозавр, Гигантозавр. События: `training` ($3000, +★ **любому** своему динозавру с любой локации), `experiment` ($3000; случайно интеллект/+★, агрессия/+★ или смерть — **любой** свой динозавр с любой локации), `money` с `amount: 3000`, углы `rest` (Инкубатор / Показатели / Выходной). Бонусы ★ — в `campaign.attrBonusByLocation`.

Размер круга = `location.cells.length` (`getCellCount`), не всегда 40. Отрисовка — `boardLayout` / `buildSideMap` в `board.js`.

У каждого игрока свой `locationId`: можно быть на разных локациях одновременно. Вкладка локации переводит **только текущего** (офлайн) / **себя** (онлайн). Доска показывает локацию текущего хода и фишки только тех, кто там же. Собственность, позиции и бонусы ★ — в `game.campaign` по локациям; деньги общие.

## 4. Архитектура (как устроено)

### 4.1 Слои
1. **Данные** (`locations.js`) — чистые описания клеток, без UI.
2. **Движок** (`game.js`) — мутирует объект `game`, пишет лог, проверяет победу.
3. **Вид** (`board.js`) — DOM поля из location + players + ownership.
4. **Приложение** (`app.js`) — экраны, биндинги, persist, онлайн-синк.
5. **Сеть** (`auth.js` + `net.js`) — опциональный слой поверх движка.

### 4.2 Модель состояния `game` (ключевые поля)
- `location`, `players[]`, `currentPlayerIndex`, `canRoll`
- `ownership` — `{ cellIndex → playerId }` на текущей локации
- `pendingAction` — `buy` | `sell` | `jail` | `restaurant` | `training` | `experiment` | null (блокирует кубик)
- `pendingTrade` — обмен 1↔1 купленными динозаврами между игроками с любых локаций (`offerLocationId` / `requestLocationId`; кубик **не** блокирует). Свободных обменивать нельзя.
- `pendingBattle` — бой (вызов → выбор дино → раунды атака/щит → aftermath). Кубик **не** блокирует. Статы: HP/урон от size/fame; черты `intellect`/`aggression` в `campaign.traitsByLocation`; HP/смерть в `campaign.dinoHpByLocation`. Реанимация = 50% цены.
- `campaign.ownershipByLocation` / `positionsByLocation` / `attrBonusByLocation` / `traitsByLocation` / `dinoHpByLocation`
- `finished`, `winnerId`, `log[]`
- онлайн: `online`, `roomId`, `roomCode`, у игроков `userId`

### 4.3 Экраны UI (`app.js` → `state.screen`)
`boot` → `auth` → `main-menu` → `room-lobby` | `game` (режимы `offline` / `tutorial` / `online`)

## 5. Правила механик (краткий справочник)

| Константа | Значение | Где |
|-----------|----------|-----|
| Старт денег | $15 000 | `START_MONEY` |
| Цель | $125 000 | `WIN_GOAL` |
| Клеток | обычно 40; Сан-Альбертус 16; Сенегал 14 | `getCellCount(game)` |
| Круг (старт) | +$2000 | `onCompletedLap` |
| Цена динозавра | от ★ (`priceFromAttractiveness`) | ★30→$500 … ★100→$30 000 |
| Доход динозавра | ~10% цены (min $50) | `INCOME_RATE` |
| Продажа агенту | цена + $1000 (Soyona +$5000) | `SELL_BONUS` / `sellBonus` |
| Залог тюрьмы | $2000 | `JAIL_BAIL` |
| Игроков офлайн | 2 | `DEFAULT_PLAYERS` |
| Онлайн комната | 2–4 | `MIN/MAX_ROOM_PLAYERS` |

### Типы клеток (`CELL_TYPES`)
- `start` — старт  
- `empty` — ничего  
- `property` — динозавр (покупка, доход за круг, обмен)  
- `money` — бонус (`amount` / $500 / спец. $13000 на Мальте)  
- `character` — агент / Soyona / Ян Малкольм → продажа  
- `event` — `jail`, `mystery`, `return_start`, `fine`, `restaurant`, `training`, `experiment`  
- `corner` — тюрьма (`effect: 'fights'`) или отдых (`rest`)

### Ход
1. Бросок 1d6 → движение по кругу.  
2. При пересечении старта: +$2000 + доход с своих динозавров → `checkWin`.  
3. Эффект клетки; если `pendingAction` — ждём решения; иначе ход следующему.  
4. Тюрьма в начале хода: заплатить $2000 (и сразу бросать) или отсидеть ход.  
5. Купленный динозавр на локации недоступен другим (аренды нет — только «занято»).  
6. Обмен в любой момент (на любой открытой локации): только **купленный свой** ↔ **купленный другим игроком**, динозавры со **всех** локаций кампании. Ориентир ★: `size×4 + fame×4 + dietScore×2 + (flies?5:0)`. Рацион: растительноядный / плотоядный / всеядный / рыбоядный; у летающих `flies: true`.

### Dev-режим
`Ctrl+Shift+D` / кнопка Dev: телепорт, forced dice, деньги, jail, смена активного игрока.

## 6. Онлайн (Supabase)

1. Выполнить `sql/schema.sql`.  
2. Вписать URL и anon key в `js/config.js`.  
3. Email auth включён.  
4. Хост создаёт комнату → invite `?room=CODE` → старт при ≥2 игроках.  
5. Состояние пушится через `JMNet.pushGameState` после действий; подписка Realtime на `games`.  
6. Ходы онлайн: только игрок с `userId` на текущем сиденье (`isMyOnlineTurn`).

RLS: профили/комнаты/игроки/игры — authenticated; update games — участник комнаты.

## 7. Найденные особенности и риски

1. **Нет аренды** при попадании на чужого динозавра — только отказ в покупке.  
2. **Сен-Юбер** без клетки тюрьмы `fights` — `findPrisonCellIndex` fallback на индекс 10.  
3. **Конфликты онлайн**: last-write-wins по JSON state (нет операционного transform).  
4. **Dev-действия** в `app.js` иногда делают `renderAll` без `persistState` — в онлайне могут не уехать на сервер.  
5. **Статика без bundler** — держать порядок `<script>` и зависимости через `window.*`.  
6. Картинки в `img/` (в разметке есть ссылки; каталог может отсутствовать в workspace — UI терпит).  
7. Стиль кода: IIFE, без модулей ES; UI на русском.

## 8. Правила агента для этого репозитория

1. При старте сессии — читать **этот файл**.  
2. Отвечать пользователю **по-русски**.  
3. Сохранять ванильный стек (не навязывать React/Vite без явной просьбы).  
4. Игровую логику не дублировать в `app.js` — только в `game.js`.  
5. Новые локации — через `locations.js` (`makeCells(list, n)`, при n≠40 задать `boardLayout: { sideH, sideV }` где `4+2*sideH+2*sideV = n`).  
6. Онлайн-изменения согласовывать с `serializeGameState` / `hydrateGameState` и схемой SQL.  
7. Не коммитить секреты; `config.js` — плейсхолдеры, не реальные ключи.  
8. Существенные изменения механик/архитектуры — отразить в этом файле.  
9. Не трогать git/config без запроса пользователя.  
10. UI: не ломать существующий визуальный язык (тёмные темы локаций, Russo One / Nunito).
